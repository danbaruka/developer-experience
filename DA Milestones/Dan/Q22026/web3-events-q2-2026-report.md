# Web3 Events Q2 2026: Open Source Summit and Safrochain Community Call

**Author:** Dan Baruka, Developer Advocate, Developer Experience Working Group  
**Quarter:** Q2 2026  
**Events attended:** 2

## Overview

In Q2 2026 I attended two Web3 events: a global open source conference and a community call focused on real-world financial use cases. Both aimed to share Cardano developer tooling and gather feedback from builders.

## Event 1: Open Source Summit North America 2026

**Event:** [Open Source Summit North America](https://events.linuxfoundation.org/open-source-summit-north-america/)  
**Dates:** May 18 to 20, 2026  
**Location:** Minneapolis, Minnesota (attended virtually)  
**Host:** Linux Foundation

I presented virtually on **setting up Yaci DevKit**, a local Cardano development environment with a devnet, Blockfrost-compatible indexer (Yaci Store), and browser explorer (Yaci Viewer).

The demo covered:

- Why a local devnet helps smart contract and dApp development
- Step-by-step Yaci DevKit setup
- How Yaci Store works with SDKs like Blaze and Mesh
- Moving from local devnet to public testnet

**Key takeaway:** Developers respond well to local-first setups they can run on a laptop without cloud accounts or API keys. Several attendees asked about Blockfrost-compatible APIs and Cardano transaction building, which aligns with ongoing DevEx documentation work.

## Event 2: Safrochain Community Call

**Event:** Safrochain Community Call  
**Format:** Virtual  
**Focus:** Real-world financial use cases on blockchain

This call explored how blockchain applies to everyday situations: cross-border payments, community savings, and support for local economies.

Topics discussed:

- Cross-border remittances and settlement
- Community savings with on-chain transparency
- Local economic use cases for merchants and cooperatives
- Builder challenges: SDK choice, wallet integration, and testnet to production paths

**Key takeaway:** Community calls surface practical onboarding needs that surveys often miss. Builders care more about implementing real flows (savings, remittances) than choosing between many SDKs upfront.
